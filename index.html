tinyform
any document.
any schema.
pure JSON.

document extraction infrastructure. send a document, define a schema, receive structured JSON.

get started

install the client:

pip install tinyform

pass a document and a schema. get JSON back.

# initialize once
tf_client = tinyform(api_key="tf_...", base_url="https://api.tinyform.dev")

# pass any document and a target schema
result = tf_client.get("invoice.pdf", {
    "vendor_name": "string",
    "total_amount": "float",
    "due_date": "date",
    "line_items": [{"description": "string", "qty": "int"}]
})

the output will match the schema you provide. constrained decoding means the response structure is determined by your schema at generation time.

use cases

kafka pipeline
# documents arrive as stream events
for message in kafka_consumer.poll():
    result = tf_client.get(message.value["path"], schema)
    kafka_producer.send("structured-docs", result)
fields inferred from context
result = tf_client.get("contract.pdf", {
    "parties": ["string"],
    "contract_value": "float",
    "sentiment": "enum[positive, neutral, negative]",
    "risk_flag": "bool",       # inferred from language
    "urgency": "enum[low, medium, high]"  # inferred from tone
})

schema fields don't have to appear literally in the document. the model can infer values like sentiment or urgency from the language and context of the text.

batch processing
results = [tf_client.get(str(doc), schema) for doc in Path("./inbox").glob("*")]
pd.DataFrame(results).to_csv("processed.csv")
healthcare documents
result = tf_client.get("prior_auth.pdf", {
    "patient_name": "string",
    "diagnosis_codes": ["string"],
    "requested_procedure": "string",
    "urgency": "enum[routine, urgent, emergent]"
})

about

tinyform extracts structured JSON from unstructured documents. you define the schema, send a document, and receive a JSON object that matches that shape.

it runs on a small 1-bit model fine-tuned for document extraction, served from owned hardware. the output format is enforced at generation time through constrained decoding.

the model understands language, not just document structure. schema fields can include inferred values — sentiment, urgency, risk signals — that aren't explicitly stated in the source document.
